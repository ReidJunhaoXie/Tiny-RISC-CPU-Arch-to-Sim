## 2. 指令暫存器(IR)與狀態機

- **state=0：取指階段（Fetch）**
    - 目的：從RAM讀取指令，並寫入指令暫存器（IR）。
    - 主要訊號設定：
        - `en_ir = 1`（開啟IR寫入）
        - `en_pc = 1`（PC加1，往下一條指令）
        - `ld = 1`（RAM輸出enable，讀出資料）
        - `halt = 1`（system clock啟用）
        - ALU暫存器disable（en_a=0）
        - 其他控制訊號可設為 `x`（don’t care）
    - PC於此階段自動累加、取下一條指令，ALU與資料不運作。

- **state=1：執行階段（Execute）**
    - 目的：根據IR內容與LUT輸出控制訊號執行具體操作（計算、寫回、跳轉等）
    - 主要訊號設定：
        - `en_ir = 0`（關閉IR寫入）
        - `en_pc = 0`（除非執行跳轉，PC暫停不動）
        - `ld`、`str` 按照指令
        - `en_a = 1`（ALU暫存器enable，可寫入計算結果）
        - 其餘如`jmp_en`、`je_en`、`en_cmp`等訊號由LUT控制
    - 如果執行跳轉指令（jmp或je且cmp=1），則PC的`ld`使能啟動，由IR中的目標地址直接寫入PC，完成跳轉。

## 3. LUT 重建

- **LUT擴充內容**：
- 使用一個D-Flip-Flop構成的狀態機（取指=0，執行=1），clock每個上升沿自動交替切換，簡化時序控制。
- 新狀態控制訊號如 en_ir、cmp、en_pc、ld_pc... 由LUT自動產生
  - 每個opcode（含state/cmp等組合）對應所有需用到的控制訊號
  - 原本compare外部電路整合進LUT邏輯
  - LUT的輸入才可以真正代表整個電路的行為，是整個系統的控制「system control code」，system control code = opcode + cmp + state，state系統自行發出、cmp always on，兩者都不是外部控制的
