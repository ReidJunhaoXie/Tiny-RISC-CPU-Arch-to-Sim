# Von Neumann 計算機架構
## 1.系統
1. Memory - 存程式指令跟資料的地方
2. ALU(Arithmatic Logic Unit) - 算術運算、邏輯運算電路
3. Control Unit - 控制單元，取址、解碼、產生控制訊號(Mux)
4. I/O - 輸出輸入
### Memory
程式指令跟資料存放的位置，也就是說程式指令跟資料實際上有相同的格式。
### 指令的執行
程式-即一組數量有限的指令，控制單元取址、解碼、執行，指令會逐條執行。

CPU中的暫存器、CPU中的Cache、主記憶體，主要區別在:
1. 原理 : CPU Reg的基本組成是D-flipflop，最快；CPU Cache的元件是SRAM，內部同樣是電晶體；DRAM 的儲存元件是一個電容+一個電晶體，有操作限制，需要刷新高電位，也被電容的充放電限制入肚
2. 位置 : 三者具體CPU的位置 -> CPU Reg - CPU Cache - DRAM
3. 容量 : CPU Reg < CPU Cache < DRAM

### 程式的執行
程式 -> 指令 -> 執行 
通過指令將輸入的資料變成想要的輸出。
- Machine Period
取指令 - 解碼 - 執行 - 取址 - ... - 無指令

#### 取指令(fetch)
CPU的控制單元把下一條待執行的指令複製到CPU的(IR)Instruction Register，該指令地址存到PC(Program Counter)，接著PC+1，指向下一條待執行指令
#### 解碼(decode)
IR得到指令後，控制單元(control unit)會負責解碼，產生二進位的代碼。
#### 執行(execute)
解碼完成後，控制單元發送命令到CPU的，ex:從記憶體讀取資料到CPU

### CISC vs RISC
- CISC : 複雜指令級，底層需支援大量的指令，因此硬體實現難度高，也難以做硬體的最佳化，代表人物是intel的x86，但現在實際使用中，Intel的CPU底層加入了解碼器 ! 把複雜的指令再拆成精簡的指令，所以CISC也隱含RISC的思想
- RISC : 精簡指令集，盡可能用最少的指令做到所以該支援的操作 ! 底層硬體需要支援的操作少，因此更易做到優化，但同時增加了程式設計的難度。
### Pipeline
如果控制單元可以同時控制複數訊號，下一條指令就可以再當前指令完成前開始，如今的CPU多是這樣運行的，如以下 : 
1. execute - fetch - decode - execute - fetch - decode -
2. decode - execute - fetch - decode - execute - fetch - 
3. fetch - decode - execute - fetch - decode - execute -
